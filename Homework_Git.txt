1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?

Команда git checkout



2.Как вернуть файл из индекса в рабочий каталог?

Команда git restore --staged имя файла



3.Как отменить ошибочный commit ?

Команда git reset HEAD~1(если вернуться на шаг назад)




4.Как создать новую ветку и переключится в нее?

Команда git branch название ветки




5.В чем отличение между fast-forward merge  and non fast-forfard merge?

Отличие между fast-forward merge и non fast-forward merge  в том, как Git объединяет изменения из двух веток и какая история коммитов формируется после слияния.

Fast-forward merge применяется, когда ветка, в которую мы сливаемся, находится впереди текущей ветки по истории коммитов. 

Non fast-forward merge применяется, когда ветка, в которую мы сливаемся, имеет новые коммиты, которых нет в текущей ветке. 





6.Когда может возникнуть конфликт слияния веток, как  его разрешить?

Если мы и кто-то еще редактировали один и тот же файл.Либо же мы у себя в двух разных ветках изменили один и тот же файл.
Когда возникает конфликт Git отмечает файл специальными метками, такие как <<<<<<<<HEAD, ========, >>>>>>> other.branch.
Чтобы разрешить конфликт мы должны открыть файл в текстовом редакторе и удалить метки.Сохранить файл, Пометить его, как разрешенный конфликт слияния командой got add . и после этого завершить процесс слияния с помощью команды git commit.




7.Можно ли удалить ветку? Как? 

Сначала нужно уйти с этой ветки git checkout имя ветки. Затем удаляем командой git branch -d имя ветки. Если на ветки есть файлы, но они нам не нужны, то команда git branch -D имя ветки
